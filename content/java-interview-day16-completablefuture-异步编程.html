📌 CompletableFuture 异步编程

进度 24%（16/66）
📖

核心概念

CompletableFuture 是 Java 8 引入的异步编程工具类，实现了 Future 和 CompletionStage 接口，支持函数式编程和链式调用。

核心优势：

  • 异步非阻塞：任务在后台线程执行，主线程不阻塞
  • 链式编排：支持 thenApply、thenAccept、thenCompose 等链式调用
  • 异常处理：内置 exceptionally、handle 等异常处理机制
  • 结果合并：支持 thenCombine、allOf、anyOf 等多任务组合

与 Future 接口的区别：

  • Future 只能通过 get() 阻塞获取结果
  • CompletableFuture 支持回调函数，结果就绪时自动触发
  • CompletableFuture 支持任务流水线编排
⚙️

创建方式

1. supplyAsync / runAsync

// 有返回值
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> "result");

// 无返回值
CompletableFuture<Void> future = CompletableFuture.runAsync(() -> System.out.println("task"));

// 指定线程池（推荐，避免共用ForkJoinPool）
ExecutorService executor = Executors.newFixedThreadPool(10);
CompletableFuture.supplyAsync(() -> "result", executor);

2. 已完成的 Future

CompletableFuture<String> future = CompletableFuture.completedFuture("ok");

⚠️ 注意：不指定线程池时，默认使用 ForkJoinPool.commonPool()，整个 JVM 共享，线程数=CPU核心数，IO密集型场景务必自定义线程池！

⚖️

链式回调 API

方法入参返回值说明
thenApplyFunctionU转换结果，有返回值
thenAcceptConsumerVoid消费结果，无返回值
thenRunRunnableVoid不关心结果，执行后续
thenComposeFunctionCompletionStage扁平化，依赖上一个结果
thenCombineBiFunctionU合并两个独立 Future 的结果

示例：

CompletableFuture.supplyAsync(() -> 100)
    .thenApply(i -> i * 2)          // 转换: 200
    .thenAccept(i -> log(i))        // 消费
    .thenRun(() -> clean());        // 收尾
🔍

多任务组合

allOf - 等待所有完成

CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> "A");
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> "B");
CompletableFuture<String> f3 = CompletableFuture.supplyAsync(() -> "C");

// 等所有完成，无返回值
CompletableFuture.allOf(f1, f2, f3).join();

// 获取所有结果
String r1 = f1.get(), r2 = f2.get(), r3 = f3.get();

anyOf - 任一完成即返回

CompletableFuture<Object> any = CompletableFuture.anyOf(f1, f2, f3);
Object firstResult = any.get();

applyToEither - 取最快结果并转换

f1.applyToEither(f2, s -> s + " wins!");

使用场景：超时降级设计——同时调用缓存和DB，哪个先返回用哪个。

🎯

异常处理

1. exceptionally - 异常恢复

CompletableFuture.supplyAsync(() -> 10 / 0)
    .thenApply(i -> i * 2)
    .exceptionally(ex -> {
        log.error("error", ex);
        return 0;  // 默认值降级
    });

2. handle - 无论成功失败都处理

.handle((result, ex) -> {
    if (ex != null) return "fallback";
    return result;
});

3. whenComplete - 副作用，不修改结果

.whenComplete((result, ex) -> {
    if (ex != null) monitor.incError();
    else monitor.incSuccess();
});

⚠️ 面试重点：如果不处理异常，异常会被静默吞噬！建议所有 CompletableFuture 链尾都加异常处理。

⚠️

生产踩坑经验

坑1：默认线程池被耗尽

IO密集型任务用默认 ForkJoinPool（线程数=CPU核数），导致所有异步任务阻塞。解决：业务专用线程池。

坑2：异常被吞导致排障困难

没有 exceptionally/handle 时，异常不会打印，任务看起来卡死。解决：统一加 whenComplete 打日志。

坑3：get() 无限阻塞

Future.get() 不带超时，任务异常或卡住会永久阻塞主线程。解决：永远用 get(timeout, unit)，或用 join() 但配合超时。

坑4：线程上下文丢失

ThreadLocal 在异步线程中丢失（如用户登录态）。解决：调用前捕获上下文，在回调中恢复；或用自定义线程池包装 Runnable。

坑5：OOM 风险

循环创建大量未完成的 CompletableFuture 导致内存泄漏。解决：加限流，控制并发数。

❓

面试真题

Q1: CompletableFuture 和 Future 有什么区别？

A: Future 是 Java 5 接口，只能通过阻塞 get() 获取结果，不支持回调、链式编排、异常处理。CompletableFuture 是其实现类，支持函数式编程、多任务组合、非阻塞回调。

Q2: supplyAsync 和 runAsync 的区别？

A: supplyAsync 是 Supplier 有返回值，runAsync 是 Runnable 无返回值。

Q3: thenApply 和 thenCompose 的区别？（高频）

A: thenApply 是 Function<T,U>，返回 U，结果会被包装成 CompletableFuture；thenCompose 是 Function<T,CompletionStage<U>>，直接返回 CompletableFuture，用于避免嵌套（CompletableFuture<CompletableFuture<U>>）。类似 map vs flatMap。

Q4: allOf 有返回值吗？如何获取所有结果？

A: allOf 返回 CompletableFuture<Void>，无返回值。需要在 allOf.join() 后分别调用每个 future 的 get()。

Q5: CompletableFuture 是线程安全的吗？

A: 是。内部用 CAS 操作维护状态，多线程同时 complete 也不会有并发问题。
