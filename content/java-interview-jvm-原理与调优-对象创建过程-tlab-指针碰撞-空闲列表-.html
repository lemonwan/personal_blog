19 对象创建过程（TLAB、指针碰撞/空闲列表）

📖对象创建全流程：从 new 到内存分配

在 Java 中，一个对象从代码 new Object() 到真正 usable，要经历以下完整流程：

  1. 类加载检查 — JVM 遇到 new 指令时，首先会去检查这个指令的参数能否在 常量池 中定位到一个类的符号引用，并对该类进行加载、校验、准备、解析、初始化（如果还没做过）。
  2. 内存分配 — 类检查通过后，JVM 需要在堆中划分一块确定大小的内存空间。对象所需内存大小在类加载完成后即可确定。
  3. 初始化零值 — 分配到的内存空间都初始化为零值（不包含对象头），保证对象的实例字段在 Java 代码中可以不赋初值就直接使用。
  4. 设置对象头 — 进行对象头的设置，包括 哈希码（@Object.hashCode）、GC 分代年龄、锁状态标志、线程持有的锁、偏向线程 ID、偏向时间戳等。
  5. 初始化（执行 <init>） — 执行 <init> 方法，按照程序流程进行属性的赋值和代码块执行，完成对象的真正初始化。

🔒指针碰撞 vs 空闲列表：堆内存分配策略

JVM 在堆中分配内存时，根据堆的规整程度选择不同的分配方式：

📌 前提
堆内存必须是规整的，即已使用的内存块集中在某一区域，空闲的在另一区域，中间放一个指针作为分界。
分配方式适用堆类型原理优点缺点
指针碰撞（Bump the Pointer）规整的堆（如 Serial、ParNew 收集器）使用一个指针，分配时将指针向空闲方向移动一段对象大小的距离分配速度快，只需一次指针移动前提是堆必须规整
空闲列表（Free List）不规整的堆（如 CMS 收集器）虚拟机维护一个记录空闲内存块的列表，分配时从列表中找到足够大的空间划分适用于碎片化严重的堆分配时需要遍历列表，性能略低
⚠️ 并发安全问题
多个线程同时创建对象时，指针碰撞或修改空闲列表都需要 CAS + 失败重试 或 TLAB 来保证线程安全。

🔬TLAB（Thread Local Allocation Buffer）线程私有的分配缓冲区

TLAB 是 Thread Local Allocation Buffer 的缩写，是 JVM 为每个线程在 Eden 区分配的线程私有的分配缓冲区。

  • 目的：避免多个线程在同一 Eden 区域分配对象时产生并发竞争，消除 CAS + 失败重试 的开销。
  • 工作原理：每个线程在 Eden 中拥有一块小的缓冲区，线程在自己 TLAB 内分配对象时不需要加锁，只需移动指针即可。
  • 分配规则：当 TLAB 空间不足时，JVM 要么重新分配一块 TLAB，要么直接在 Eden 上通过 CAS 竞争分配。

TLAB 不是强制的，默认开启（-XX:+UseTLAB），也可以通过参数关闭。

为什么不用 100% TLAB 分配？ 因为对象的实际大小可能大于 TLAB 剩余空间，此时需要在 Eden 上重新分配；另外 TLAB 空间很小（默认 Eden 的 1%），很多情况下仍然需要全局分配。

🔒对象内存布局与 OOP-Klass 模型

对象在内存中的布局分为三块（HotSpot VM）：

区域说明
对象头（Header）包含两部分：① Mark Word：存储哈希码、GC 年龄、锁信息等；② Klass Pointer：指向类元数据的指针（在 64 位开启指针压缩时为 4 字节）
实例数据（Instance Data）对象真正有效的信息，包括父类字段和自身字段，需要对齐填充
对齐填充（Padding）HotSpot 要求对象起始地址必须是 8 字节的整数倍，不足的用零值填充
📌 OOP-Klass 模型
HotSpot 使用 OOP-Klass 模型管理对象类型信息：
• OOP（Ordinary Object Pointer）：对象头中的 Klass Pointer，指向 Klass 元数据
• Klass：描述类的元数据（字段、方法等），存储在 元空间（Metaspace） 中
• 这种设计将对象实例和类元数据分离，便于 对象压缩指针（Compressed Oops） 优化

🎤面试官常考：对象分配相关的深度问题

  • Q: 对象一定在堆上吗？ — 不一定，JIT 编译后的 逃逸分析 可能将对象栈上分配；标量替换 后甚至不上内存。
  • Q: 为什么用 B+ 树做索引却用指针碰撞分配内存？ — 索引需要稳定结构用 B+ 树，内存分配追求速度，规整堆上指针碰撞最快。
  • Q: TLAB 满了怎么办？ — 尝试分配新的 TLAB；如果仍然放不下，直接在 Eden 的共享区域通过 CAS 分配。
  • Q: 开启指针压缩后对象头有什么变化？ — Mark Word 大小不变（8 字节），Klass Pointer 从 8 字节压缩为 4 字节。
  • Q: 如何查看对象的内存布局？ — 使用 -XX:+PrintLiveDump 或在 JVM 启动时开启 -XX:+UnlockDiagnosticVMOptions -XX:+PrintOOMemoryLayout。
