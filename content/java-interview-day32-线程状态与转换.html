核心概念

Java 线程在生命周期中存在 6 种不同的状态，特定条件下会相互转换。这是并发编程的基础，也是面试高频考点。

线程的 6 种状态（Thread.State 枚举）：

  • NEW（新建）：线程对象已创建，但尚未调用 start() 方法
  • RUNNABLE（可运行）：Java 虚拟机中正在执行或等待 CPU 时间片
  • BLOCKED（阻塞）：等待获取监视器锁（synchronized）
  • WAITING（等待）：无限期等待另一个线程执行特定动作（需被唤醒）
  • TIMED_WAITING（计时等待）：指定等待时间，超时自动返回
  • TERMINATED（终止）：线程执行完毕退出

❷ RUNNABLE 状态的特殊性

Java 的 RUNNABLE ≠ 操作系统中的 Running。它包含两种情况：

  • Running：正在 CPU 上执行
  • Ready：就绪队列中等待调度

JVM 不区分这两种子状态，因为线程切换太快，JVM 认为它们是同一状态。

底层原理深入

HotSpot 中的线程状态映射：

OSThread 维护 _thread_state 变量，JVM 将其映射为 Java Thread.State：

  • NEW：线程对象在 Java 堆中，OS 线程尚未创建
  • RUNNABLE：调用 os::start_thread()，内核线程启动
  • BLOCKED：进入 ObjectMonitor 的 _cxq 或 EntryList
  • WAITING：在 ObjectMonitor 的 WaitSet 中 park

park/unpark 机制（LockSupport）：

JDK 1.5+ 引入的底层线程挂起/唤醒机制，比 wait/notify 更灵活：

  • 支持先 unpark 再 park（无唤醒丢失问题）
  • 不需要锁对象，可随时调用
  • 是 AQS、LockSupport 等的基础

面试真题要点

【真题1】调用 start() 和 run() 的区别？

  • start()：启动新线程，进入 RUNNABLE，由 JVM 调度执行 run()
  • run()：普通方法调用，在当前线程执行，不启动新线程

【真题2】如何强制线程状态dump？

jstack <pid>  # 查看所有线程状态
kill -3 <pid>  # 输出线程dump到控制台
Thread.getAllStackTraces()  # 代码中获取

【真题3】线程可以多次 start() 吗？

不能。一旦启动，线程状态不再是 NEW，再次调用 start() 会抛出 IllegalThreadStateException。这是不可逆的状态转换。

【真题4】WAITING 状态的线程如何被唤醒？

  • notify() / notifyAll()
  • 线程被中断（interrupt()）
  • 虚假唤醒（Spurious Wakeup，罕见但必须处理）

实际运用场景

场景1：线程池监控

通过 ThreadPoolExecutor 的 getActiveCount()、getTaskCount() 监控线程状态，及时发现线程阻塞。

场景2：死锁检测

大量 BLOCKED 状态线程 + 循环等待 → 死锁特征。jstack 自动检测死锁信息。

场景3：性能调优

  • 大量 TIMED_WAITING：检查锁粒度是否过大
  • 线程卡在 WAITING：检查是否有 notify 丢失
  • RUNNABLE 但无CPU消耗：可能在阻塞IO（网络/磁盘）

场景4：优雅停机

executor.shutdown();
if (!executor.awaitTermination(60, SECONDS)) {
    executor.shutdownNow(); // 中断正在执行的任务
}
⚠️

踩坑经验

💣 坑1：TIMED_WAITING 不是释放锁的信号

sleep(time) 不释放锁！wait(time) 才释放锁。用 sleep 控制并发是常见错误。

💣 坑2：虚假唤醒

WAITING 状态的线程可能在没有 notify 的情况下醒来。必须在循环中检查条件：

while (条件不满足) {
    wait();  // 不能用 if！
}

💣 坑3：interrupted() 清除中断标记

Thread.interrupted() 是静态方法，返回后会清除中断标记。而 isInterrupted() 仅查询不清除。

💣 坑4：线程 TERMINATED 后无法复活

线程一旦终止，无法再次启动。需要复用线程请使用线程池，不要自己再造轮子。

Q&A 速记

Q: 线程状态有几种？分别是什么？
A: 6种：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED

Q: wait() 和 sleep() 对锁的影响？
A: wait() 释放锁，sleep() 不释放

Q: RUNNABLE 就是正在运行吗？
A: 不一定，还包括就绪等待调度的状态

Q: 一个线程两次调用 start() 会怎样？
A: 抛出 IllegalThreadStateException

Q: 如何查看线程状态？
A: jstack、jconsole、VisualVM、Thread.getState()
