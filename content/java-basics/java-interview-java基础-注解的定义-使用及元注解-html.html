08 注解的定义、使用及元注解

📖注解的基础概念

  • 注解（Annotation）是 Java 5 引入的元数据机制，可以为代码添加额外信息，且不影响代码执行
  • 注解通过 @interface 关键字定义，本质上是一种特殊的接口
  • 常见内置注解：@Override、@Deprecated、@SuppressWarnings、@FunctionalInterface
  • 注解可以标注在：类/接口、方法、字段、参数、局部变量、包、类型使用 等位置

📖自定义注解的定义方式

  • 使用 @interface 定义自定义注解：
public @interface MyAnnotation { String value(); // 必须指定的属性 String name() default ""; // 带默认值的属性 int priority() default 1; // 数值类型属性 Class<?> type() default Void.class; // 类型属性 Status level() default Status.NORMAL; // 枚举属性 String[] tags(); // 数组类型属性 } // 使用方式 @MyAnnotation(value = "test", name = "demo", tags = {"a", "b"}) public void myMethod() {}
  • 注解属性类型只能是：基本数据类型、String、Class、Enum、Annotation 以及它们的数组
  • 属性必须通过 default 设置默认值，或者在使用时必须指定
  • 如果注解只有一个属性且名为 value，使用时可以省略 value=

🔒四大元注解（Meta-Annotation）

  • @Target — 限定注解可以应用的目标位置：
@Target({ElementType.TYPE, ElementType.METHOD}) public @interface MyAnnotation {} // ElementType.METHOD — 方法 // ElementType.FIELD — 字段 // ElementType.TYPE — 类/接口/枚举 // ElementType.PARAMETER — 参数 // ElementType.CONSTRUCTOR — 构造器
  • @Retention — 定义注解的保留策略（生命周期）：
@Retention(RetentionPolicy.SOURCE) // 仅源码阶段，编译后丢弃 @Retention(RetentionPolicy.CLASS) // 编译期保留，默认值 @Retention(RetentionPolicy.RUNTIME) // 运行期可获取，最常用
\U0001F4CD 面试要点：只有 @Retention(RetentionPolicy.RUNTIME) 的注解才能通过反射在运行时获取
  • @Documented — 标注注解是否会被 javadoc 提取到文档中
  • @Inherited — 标注注解是否可以被子类继承（仅对类有效）

🔬运行时注解的反射处理

  • 通过反射获取注解信息，是注解的核心使用方式：
// 1. 判断某个方法是否有指定注解 method.isAnnotationPresent(MyAnnotation.class) // 2. 获取注解实例 MyAnnotation annotation = method.getAnnotation(MyAnnotation.class); // 3. 读取注解属性值 String value = annotation.value(); String name = annotation.name(); // 4. 在类级别遍历所有带注解的方法 for (Method m : clazz.getDeclaredMethods()) { if (m.isAnnotationPresent(MyAnnotation.class)) { MyAnnotation ann = m.getAnnotation(MyAnnotation.class); // 执行业务逻辑 } }
常用反射 API：
isAnnotationPresent() — 判断是否标注
getAnnotation() — 获取指定注解实例
getAnnotations() — 获取所有注解
getDeclaredAnnotations() — 获取直接声明的注解

🛠️注解的实际应用场景

  • Spring 框架： @Autowired（依赖注入）、@Component（组件注册）、@RequestMapping（URL映射）等，都依赖 @Retention(RUNTIME) + 反射扫描
  • Junit 测试： @Test、@Before、@After 等，基于 @Retention(SOURCE) 或 RUNTIME
  • Lombok： @Data、@Builder 等，在编译期通过 @Retention(SOURCE) + APT（注解处理器）自动生成代码
  • Retrofit / RxJava： @GET、@POST 等 HTTP 注解，运行时通过反射构建请求
  • 自定义注解处理器（APT）： 继承 javax.annotation.processing.AbstractProcessor，在编译期解析注解并生成 Java 代码
\u26A0\uFE0F 注解处理器（APT）与运行时注解的区别：
• APT：编译期执行，生成源码 → 需要继承 AbstractProcessor
• 运行时注解：运行期通过反射获取，需要 @Retention(RUNTIME)

❓面试高频问题

  • Q：注解和注释有什么区别？
    A：注解是代码的元数据，可以被工具/框架处理；注释是给开发者看的，编译器会忽略
  • Q：@Target 和 @Retention 的作用是什么？
    A：@Target 控制注解能用在哪些位置；@Retention 控制注解的生命周期
  • Q：如何自定义一个注解？
    A：使用 @interface 定义，配合元注解 @Target、@Retention 等，定义属性并通过 default 设置默认值
  • Q：注解能继承吗？
    A：普通注解不能直接继承；但 @Inherited 元注解可以让类级别的注解被子类继承
  • Q：Spring 如何扫描并处理注解？
    A：Spring 启动时通过反射扫描 classpath 下的类，检查是否标注了特定注解，然后进行相应的处理（如注册 Bean、建立映射关系等）
