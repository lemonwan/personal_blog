14 HashSet/TreeSet 底层原理

📖HashSet 底层原理

HashSet 底层基于 HashMap 实现。

  • 底层使用 HashMap<Object, Object>，每个元素作为 HashMap 的 key 存储，value 固定为一个 static final Object PRESENT = new Object()
  • 插入元素时，先计算 hash 值，再通过 HashMap 的 put 方法存入
  • 不允许重复元素，因为 HashMap 的 key 不允许重复
  • 不保证元素的迭代顺序（无序集合）

核心方法映射：

  • add(e) → map.put(e, PRESENT)
  • remove(e) → map.remove(e)
  • contains(e) → map.containsKey(e)

🔒TreeSet 底层原理

TreeSet 底层基于 TreeMap 实现，是有序集合。

  • 底层使用 TreeMap<E, Object>，元素作为 TreeMap 的 key 存储
  • 基于 红黑树（自平衡二叉查找树） 实现
  • 元素按照自然顺序排序（实现 Comparable 接口）或自定义 Comparator 排序
  • 支持 ceiling()、floor()、higher()、lower() 等导航方法
  • 不允许重复元素（key 不可重复）

时间复杂度：

  • 插入/删除/查询：O(log n)
  • 遍历（中序遍历）：O(n)，元素按升序排列

⚖️HashSet vs TreeSet 对比

特性HashSetTreeSet
底层数据结构HashMap（哈希表）TreeMap（红黑树）
是否有序无序有序（自然顺序/自定义）
插入/查询时间O(1) 平均O(log n)
是否允许null允许一个null不允许null
排序支持不支持支持（Comparable/Comparator）
使用场景去重、快速查找需要排序、范围查询

🎤面试常考问题

Q: HashSet 如何保证元素不重复？

A: 依赖 equals() 和 hashCode() 方法。插入时先比较 hashCode，相同再比较 equals。必须同时重写这两个方法。

Q: 为什么重写 equals 必须重写 hashCode？

A: HashMap/HashSet 先用 hashCode 定位桶位置，再用 equals 比较内容。如果只重写 equals 不重写 hashCode，两个"相等"的对象可能有不同 hashCode，导致 HashSet 认为它们是不同的元素。

Q: TreeSet 如何实现排序？

A: 通过 Comparable 接口的 compareTo() 或 Comparator 的 compare() 方法，在红黑树插入时进行比较定位。
