07 反射的性能开销、在实际框架中的应用（Spring IOC/DI）

📖反射是什么？为什么需要它？

反射（Reflection）是Java提供的运行时自省机制，允许程序在运行期间动态获取类的信息并调用其成员。

  • 可以在运行时获取类的构造方法、字段、方法、注解等所有信息
  • 支持运行时动态创建对象、调用方法、修改字段值
  • 是几乎所有Java框架（Spring、MyBatis、Jackson等）的底层基石
💡 核心能力

获取Class对象 → 反射获取成员 → 动态调用/实例化。这是反射的三步走。

🔬获取Class对象的三种方式

Class<?> clazz1 = Person.class; // 1. 类字面量（编译期检查，最推荐） Class<?> clazz2 = person.getClass(); // 2. 实例.getClass()（运行期获取） Class<?> clazz3 = Class.forName("com.example.Person"); // 3. forName（类加载器，最灵活）
  • forName 会触发类的初始化（执行静态块），而 .class 不会
  • 反射获取的Class对象在JVM中是单例的，相同类型只存在一份

🔒反射的性能开销——面试重点

反射的核心痛点：性能不如直接调用。面试官常问"反射为什么慢"和"如何优化"。

性能差距对比

方式相对性能原因
直接调用1x（基准）编译期绑定，JVM可直接优化（内联、逃逸分析）
普通反射~10x~ 慢运行时动态查找方法、安全检查、未内联
关闭安全检查后~1x~ 接近setAccessible(true) 跳过访问检查
MethodHandle~2x~ 慢Java 7+ 新API，更接近直接调用

为什么慢？三大原因

  • 动态查找：每次都要通过方法名/参数类型在运行时查找方法，无法编译期优化
  • 安全检查：默认进行权限检查（public/protected/private），消耗CPU
  • 无法内联：JIT编译器无法对反射调用做内联优化（早期版本），无法做逃逸分析和GC优化

优化手段

  • setAccessible(true)：跳过访问检查，性能提升显著
  • Cached Method/Field：缓存获取到的Method/Field对象，避免重复查找
  • MethodHandle / VarHandle（Java 7+）：更接近直接调用的反射替代品
  • CGProxy：Spring AOP等框架用CGLIB/ByteBuddy生成字节码，替代反射调用
⚠️ 面试高频

Spring中反射使用频率极高（Bean创建、AOP、属性注入），但Spring会通过MethodHandle和缓存大幅优化，所以实际框架中使用反射并不一定慢——关键看是否有缓存和优化的策略。

🛠️反射在Spring IOC中的核心应用

Spring IOC/DI几乎全程依赖反射——这是反射最大的用武之地。

Spring使用反射的场景

  • Bean实例化：通过 Class.newInstance()（旧）/ Constructor.newInstance()（新）创建对象
  • 属性注入：通过 Field.set() 注入依赖（@Autowired底层实现）
  • 方法注入：通过 Method.invoke() 调用Setter/初始化方法
  • @Bean方法调用：解析@Configuration中的@Bean注解方法并调用
// Spring 内部简化的Bean创建逻辑 public Object createBean(Class<?> beanClass) { // 1. 通过反射创建实例 Constructor<?> constructor = beanClass.getDeclaredConstructor(); constructor.setAccessible(true); // 关键优化：跳过安全检查 Object bean = constructor.newInstance(); // 2. 通过反射注入依赖 for (Field field : beanClass.getDeclaredFields()) { field.setAccessible(true); // 跳过private检查 Object dependency = resolveDependency(field); field.set(bean, dependency); // 属性注入 } return bean; }
🎯 关键优化点

Spring大量使用 setAccessible(true) 来消除反射的性能损失。同时通过 CachedIntrospectionResults 缓存Bean的信息，避免重复反射查找。

🎤面试高频问题与回答思路

  • Q: 反射和直接调用的区别？

    A: 反射是运行时动态绑定，直接调用是编译期静态绑定。反射慢但灵活，直接调用快但缺乏灵活性。

  • Q: 反射为什么慢？怎么优化？

    A: ①动态查找方法 ②安全检查 ③无法内联优化。优化：setAccessible(true)、缓存Method对象、使用MethodHandle、框架层用CGLIB/ByteBuddy生成字节码。

  • Q: Spring中哪里用了反射？

    A: Bean实例化、依赖注入(Autowired)、@Bean方法调用、AOP代理创建等。Spring通过缓存和setAccessible优化了性能。

  • Q: Class.forName 和 ClassLoader.loadClass 区别？

    A: forName会触发类初始化（执行静态块），loadClass只加载不初始化。forName内部委托给ClassLoader.loadClass但多了一步初始化。

  • Q: 反射能访问private成员吗？

    A: 可以，通过 setAccessible(true) 可以绕过访问控制检查，但SecurityManager可能会阻止。
