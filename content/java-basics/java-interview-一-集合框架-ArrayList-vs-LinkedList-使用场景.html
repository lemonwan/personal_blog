Day 3

ArrayList vs LinkedList 使用场景

Java 基础 · 集合框架 · 经典对比面试题
一-1
Java 基础 → 集合框架

1 核心概念

  • ArrayList 基于动态数组实现，底层是 Object[] elementData 数组，支持快速随机访问。初始容量 10，超过容量时自动扩容 1.5 倍。
  • LinkedList 基于双向链表实现，每个节点（Node）包含 prev（前驱）、item（数据）、next（后继）三个字段，支持高效的中间位置插入/删除。
  • 两者都实现 List、RandomAccess、Cloneable、Serializable 接口，允许 null 元素，且都不是线程安全的。
  • ArrayList 实现了 RandomAccess 接口标记为"支持随机访问"，LinkedList 没有实现此接口。

2 底层结构对比

维度ArrayListLinkedList
底层结构动态数组（连续内存）双向链表（非连续内存）
get(i) 时间复杂度O(1) 直接数组索引O(n) 需要从头/尾遍历
set(i, e) 时间复杂度O(1)O(n)
add(e) 尾部追加O(1) 均摊（扩容时 O(n)）O(1)
add(i, e) 中间插入O(n) 需 System.arraycopy 搬移元素O(1)（找到节点后）
remove(i) 中间删除O(n) 搬移后续元素O(1)（找到节点后）
空间开销较小，只存储数据 + 数组容量较大，每个节点多 2 个引用指针
内存局部性优秀（连续内存，CPU 缓存友好）较差（节点分散，缓存命中率低）

3 实战场景选择指南

  • 优先选 ArrayList：90% 以上的场景都应该使用 ArrayList。读取操作多、遍历操作多、尾部添加操作多。在现代 CPU 上，由于 CPU 缓存预取和局部性原理，ArrayList 的迭代速度通常比 LinkedList 快 5-10 倍。
  • 优先选 LinkedList：频繁在列表中间插入/删除元素的场景（如实现队列、双端队列）。LinkedList 本身实现了 Queue 和 Deque 接口，可以直接作为队列使用。
  • 面试核心结论：实际开发中 ArrayList 几乎总是更好的选择。LinkedList 所谓的"插入删除优势"仅在你已经持有目标节点引用时成立，而查找节点本身就需要 O(n)。
💡 面试要点：面试官经常追问——"你真的知道什么时候该用 LinkedList 吗？" 标准回答是：如果你的场景真的是频繁中间增删且你能用其他方式快速定位节点（如配合 Map 索引），才考虑 LinkedList。否则一律用 ArrayList。

4 底层原理深度

  • ArrayList 扩容机制：初次创建时 elementData 为空数组，第一次 add 时扩容为 10；之后每次扩容为原来的 1.5 倍（newCapacity = oldCapacity + oldCapacity/2）。扩容时用 Arrays.copyOf 创建新数组并拷贝数据。
  • ArrayList 容量预分配：可以通过 new ArrayList(expectedSize) 指定初始容量，避免频繁扩容带来的性能损耗。这在已知数据量时非常重要。
  • ArrayList.trimToSize()：将数组容量缩小到实际元素个数，释放多余内存。
  • LinkedList 内部类 Node：静态内部类，包含 Node<E> prev, E item, Node<E> next。linkFirst/linkLast/linkBefore 是核心操作方法。
  • ArrayList 的 fail-fast：通过 modCount 字段检测并发修改，迭代器调用 next() 时检查 modCount 是否变化，不一致则抛出 ConcurrentModificationException。

6 踩坑经验与最佳实践

  • 坑 #1：不知道数据量就 new ArrayList() 默认创建容量 10 的数组，如果数据量大（如 1000+），会触发多次扩容拷贝。应预判数据量指定初始容量。new ArrayList(list.size())
  • 坑 #2：在 for 循环中用 list.remove(i) 删除元素会导致漏删或越界——因为删除后后续元素前移，索引关系变化。应倒序遍历删除，或用 Iterator.remove()。
  • 坑 #3：ArrayList 的 subList() 返回的是视图而非副本，对子列表的操作会影响原列表。修改操作可能抛出 UnsupportedOperationException（如 add/remove）。
  • 坑 #4：ArrayList 在头部频繁 insert(0, e) 时性能极差（每次都要搬移全部元素），此时考虑 LinkedList 或 ArrayDeque。
  • 最佳实践：在需要队列/双端队列语义时，用 ArrayDeque 替代 LinkedList——内存占用更小、迭代更快、API 一致。
