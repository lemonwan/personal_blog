核心概念：线程创建的四种方式

Java 中创建线程主要有四种方式，每种都有其适用场景和设计考量：

  • 继承 Thread 类：最简单的方式，但 Java 不支持多继承，灵活性受限
  • 实现 Runnable 接口：解耦任务逻辑与线程控制，推荐方式
  • 实现 Callable 接口：支持返回结果和抛出检查异常，配合 Future 使用
  • 线程池：生产环境唯一正确方式，避免频繁创建销毁线程的开销
🎯

面试真题：Thread vs Runnable 区别

面试官常问："Thread 和 Runnable 有什么区别？为什么推荐使用 Runnable？"

答题要点：

  1. Java 单继承限制：继承 Thread 后不能再继承其他类，而 Runnable 是接口，可以多实现
  2. 解耦设计：Runnable 将任务逻辑与线程执行机制分离，同一 Runnable 可提交给多个线程执行
  3. 资源共享：多个线程共享同一个 Runnable 实例，便于实现数据共享
  4. 线程池兼容：Runnable 和 Callable 是线程池接受的标准任务类型
⚙️

底层原理：Callable + FutureTask

Callable 与 Runnable 的本质区别：

@FunctionalInterface
public interface Callable<V> {
    V call() throws Exception;
}

@FunctionalInterface
public interface Runnable {
    void run();
}

FutureTask 的桥梁作用：

Thread 只接受 Runnable，所以需要 FutureTask 作为适配器。FutureTask 同时实现了 Runnable 和 Future 接口，内部包装 Callable。调用 get() 方法时：

  • 任务未完成 → 阻塞等待
  • 任务完成 → 返回结果或抛出异常（包装为 ExecutionException）
  • 任务取消 → 抛出 CancellationException
💻

实际运用：生产环境只用线程池

重要警告：生产环境绝对禁止直接 new Thread！

线程池的优势：

  • 复用线程：避免线程创建销毁的昂贵开销（每个线程栈默认 1MB）
  • 控制并发数：防止系统资源耗尽（OOM: unable to create native thread）
  • 任务管理：提供任务队列、拒绝策略、监控统计
  • 生命周期管理：优雅关闭、钩子函数

创建线程池的正确姿势：

// 手动创建，不使用 Executors 工厂方法
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    corePoolSize,      // 核心线程数
    maximumPoolSize,   // 最大线程数
    keepAliveTime,     // 空闲线程存活时间
    TimeUnit.SECONDS,  // 时间单位
    new LinkedBlockingQueue<>(1000),  // 有界队列！
    new ThreadFactoryBuilder().setNameFormat("my-pool-%d").build(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
);
⚠️

踩坑经验：Executors 的隐藏陷阱

JDK 自带 Executors 工厂方法的坑：

方法问题
FixedThreadPool队列长度 Integer.MAX_VALUE，可能 OOM
CachedThreadPool最大线程数 Integer.MAX_VALUE，可能耗尽系统资源
SingleThreadExecutor同样是无界队列

真实事故：某电商系统在大促时使用 Executors.newFixedThreadPool(10)，任务队列瞬间积压数十万任务，导致 GC 频繁停顿，服务不可用。

解决方案：永远手动 new ThreadPoolExecutor，使用有界队列，自定义拒绝策略！

❓

面试延伸：线程的 start() vs run()

经典面试题："直接调用 run() 方法和调用 start() 有什么区别？"

答案：

  • start()：真正启动新线程，调用 native 方法 start0()，由 JVM 调用 run() 方法
  • run()：普通方法调用，在当前线程执行，不会启动新线程

代码验证：

Thread t = new Thread(() -> {
    System.out.println(Thread.currentThread().getName());
});
t.run();   // 输出: main （主线程执行）
t.start(); // 输出: Thread-0 （新线程执行）
