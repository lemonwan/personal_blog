07 Stream API 常用操作（map/filter/collect/reduce）

📖核心概念：什么是 Stream API？

Stream API 是 Java 8 引入的一套函数式编程风格的集合处理工具，允许以声明式方式对数据集合进行高效处理。核心思想：描述「做什么」而非「怎么做」。

💡 关键特征
  • 不存储数据：Stream 只是一个数据流的管道，不是数据结构
  • 延迟执行：中间操作（如 filter/map）惰性执行，遇到终端操作才触发
  • 单次消费：每个 Stream 只能被消费一次，用完即失效
  • 不可逆：操作不会修改源数据，产生新的 Stream

基本流程： 创建 Stream → 中间操作（零个或多个） → 终端操作（触发执行）

List<String> names = Arrays.asList("Alice", "Bob", "Charlie"); List<String> result = names.stream() .filter(name -> name.length() > 3) // 中间操作 .map(String::toUpperCase) // 中间操作 .collect(Collectors.toList()); // 终端操作 → 触发执行

🛠️四大核心操作详解

1️⃣ filter — 筛选

接收 Predicate 函数式接口，保留满足条件的元素。

stream.filter(x -> x > 0) // 保留正数 stream.filter(Objects::nonNull) // 过滤 null

2️⃣ map — 映射

接收 Function，将每个元素转换为另一种形式。

stream.map(String::length) // String → Integer stream.map(User::getName) // 对象提取字段

3️⃣ collect — 收集

将 Stream 收集为集合/Map/String 等。

stream.collect(Collectors.toList()) // 转 List stream.collect(Collectors.toSet()) // 转 Set stream.collect(Collectors.joining(", ")) // 拼接字符串 stream.collect(Collectors.groupingBy(...)) // 分组

4️⃣ reduce — 归约

将 Stream 中的所有元素通过二元操作逐步合并为一个值。

// 求和：0 + 1 + 2 + 3 = 6 int sum = numbers.stream().reduce(0, Integer::sum); // 求最大值 Optional<Integer> max = numbers.stream().reduce(Integer::max); // 累加操作：计算总价格 BigDecimal total = orders.stream() .map(Order::getAmount) .reduce(BigDecimal.ZERO, BigDecimal::add);

🔬Collectors 高阶用法

// 1. 分组统计：按状态分组 Map<Status, List<Order>> byStatus = orders.stream() .collect(Collectors.groupingBy(Order::getStatus)); // 2. 分组后计数 Map<Status, Long> countByStatus = orders.stream() .collect(Collectors.groupingBy(Order::getStatus, Collectors.counting())); // 3. 分区：满足/不满足条件 Map<Boolean, List<Integer>> partition = nums.stream() .collect(Collectors.partitioningBy(n -> n > 0)); // 4. 收集到不可变集合 (Java 16+) stream.collect(Collectors.toUnmodifiableList()); // 5. 使用 toMap 注意 key 冲突 Map<Long, User> userMap = users.stream() .collect(Collectors.toMap(User::getId, Function.identity(), (existing, replacement) -> existing)); // 保留第一个

💥🚨 常见坑与面试高频题

💥 常见陷阱
  • Stream 已被消费：重复操作同一 Stream 会抛 IllegalStateException
  • 并行流线程安全：parallelStream() 操作共享变量可能导致数据不一致
  • forEach 修改外部变量：lambda 中使用的外部变量必须为 final/effectively final
  • reduce 初始值问题：无初始值的 reduce 返回 Optional，注意空流情况
🎤 面试高频题
  • Stream 的惰性求值是如何实现的？中间操作返回新的 Stream，内部维护 Sink 链；终端操作调用 evaluate() 触发流水线执行
  • map 和 flatMap 的区别？map 是一对一映射；flatMap 解决一对多映射，将嵌套结构扁平化（如 List<List<T>> → List<T>）
  • 串行流和并行流的底层原理？并行流使用 ForkJoinPool.commonPool()，通过 Spliterator.split() 分治执行
  • peek vs forEach 区别？peek 是中间操作（延迟执行），forEach 是终端操作 — 只用于调试时不要依赖 peek 的顺序和结果
  • 为什么会说 Stream 性能比 for 循环差？小数据量下 Stream 有装箱/拆箱和 lambda 调用开销；大数据量下并行流可能优于普通 for 循环
