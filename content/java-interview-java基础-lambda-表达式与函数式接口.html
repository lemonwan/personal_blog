21 Lambda 表达式与函数式接口

📖Lambda 表达式基础

Lambda 表达式是 Java 8 引入的最重要的特性，它允许将函数作为方法参数传递，使代码更加简洁。

基本语法：

(参数列表) -> { 方法体 }

语法形式：

  • 无参数：() -> System.out.println("Hello")
  • 单参数（省略括号）：x -> x * 2
  • 多参数：(a, b) -> a + b
  • 带方法体：(x, y) -> { return x > y ? x : y; }
核心理解

Lambda 表达式本质上是一个 函数式接口的匿名实现类的实例。编译器通过类型推断和目标类型（Target Typing）来确定 Lambda 对应的函数式接口。

使用前提：Lambda 表达式必须依赖函数式接口（有且仅有一个抽象方法的接口）。

🔒四大内置函数式接口

Java 8 在 java.util.function 包中提供了四大核心函数式接口：

接口方法签名用途示例
Consumer<T>void accept(T t)消费型，有参无返回值list.forEach(System.out::println)
Supplier<T>T get()供给型，无参有返回值Supplier<Date> s = Date::new
Function<T,R>R apply(T t)函数型，有参有返回值s -> s.length()
Predicate<T>boolean test(T t)断言型，返回 booleans -> s.startsWith("A")

扩展接口：

  • BiFunction<T,U,R> - 两个输入参数
  • UnaryOperator<T> - 输入输出类型相同（Function 的子接口）
  • BinaryOperator<T> - 两个同类型输入，一个输出（BiFunction 的子接口）
  • BiConsumer<T,U> / BiPredicate<T,U>

⚖️Lambda vs 匿名内部类

对比维度Lambda 表达式匿名内部类
底层原理invokedynamic 指令，运行时动态生成编译时生成 Class$N.class 字节码文件
变量捕获只能捕获 effective final 局部变量同样要求变量是 final 或 effective final
this 指向指向外部类对象指向匿名内部类自身对象
字节码体积不生成额外的 class 文件，更轻量每个匿名内部类都会生成一个 .class 文件
使用范围只能用于函数式接口可用于任何接口或抽象类
面试高频陷阱：this 指向

Lambda 中的 this 指向外部类，而匿名内部类中的 this 指向自身实例。这个区别在很多框架的回调场景中可能引发 BUG。

🔬方法引用与构造器引用

方法引用是 Lambda 的语法糖，当 Lambda 体已有现成方法实现时，可以更简洁地引用。

四种形式：

类型语法示例等价 Lambda
静态方法引用类名::静态方法Integer::parseInts -> Integer.parseInt(s)
实例方法引用（特定对象）对象::实例方法System.out::printlnx -> System.out.println(x)
实例方法引用（任意对象）类名::实例方法String::lengths -> s.length()
构造器引用类名::newArrayList::new() -> new ArrayList()
// 示例：各种方法引用List<String> names = Arrays.asList("Alice", "Bob", "Charlie");// 1. 静态方法引用names.sort(String::compareTo);// 2. 实例方法引用（特定对象）names.forEach(System.out::println);// 3. 实例方法引用（任意对象类型）List<Integer> lengths = names.stream().map(String::length).collect(Collectors.toList());

🎤面试高频问题

Q1: Lambda 表达式的编译过程是怎样的？

javac 将 Lambda 编译为 invokedynamic 指令 + 引导方法（Bootstrap Method）-> 运行时 LambdaMetafactory 动态生成实现类 -> 后续调用直接调用已生成的实现（缓存）

Q2: 为什么 Lambda 中使用的局部变量必须是 final 或 effective final？

Lambda 捕获变量时，Java 会将变量拷贝到 Lambda 内部（值拷贝）。如果变量可以修改，拷贝的值和原值将不一致，造成数据不一致问题。因此 Java 强制要求变量是 final 或 effective final（实际不可变），保证安全。

Q3: @FunctionalInterface 注解的作用？

  • 编译检查：确保该接口有且仅有一个抽象方法
  • 文档说明：表明该接口是函数式接口，可用 Lambda 实现
  • 注意：Object 类的 public 方法（如 toString()、equals()）不算抽象方法

Q4: Lambda 与外部变量的作用域？

  • Lambda 中不允许声明与局部变量同名的参数或局部变量
  • Lambda 体中 this 指外部类对象（与匿名内部类不同）
